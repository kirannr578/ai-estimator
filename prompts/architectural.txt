==============================================================================
HARD RULE — `LS` IS FORBIDDEN AS A DEFAULT UNIT (T10 F-6)
==============================================================================

`LS` (lump sum) is the WRONG unit for almost every takeoff from an
architectural drawing. Lump-sum lines collide with the unit-mismatch
suppressor in the cost database, get zeroed in pricing, fall through to
the HAND_TAKEOFF queue, and contribute $0 to the estimate while costing
operator review time. This is the single largest driver of the
AUTO_APPROVE-share gap reported in calibration v4 (finding F-6).

You MUST NOT emit `unit: "LS"` UNLESS the line item is genuinely a
lump-sum scope with NO measurable quantity that any human estimator
could derive from the drawing or schedule. Legitimate `LS` examples:

  * "Site demolition allowance — LS"
  * "General clean-up allowance — LS"
  * "Construction layout & coordination — LS"

For EVERYTHING else — every door, window, wall, partition, fixture,
finish area, casework run, paint area, structural member, slab, footing,
piece of MEP rough-in — pick a dimensional unit (EA / LF / SF / CY / TON
/ BF / HR) from the schema enum above. Never default to `LS`.

==============================================================================
REQUIRED DIMENSIONAL UNITS BY ELEMENT TYPE
==============================================================================

Use these units. They are NOT suggestions — they are how the downstream
cost database is keyed.

| Element category                                                          | Required unit |
|---------------------------------------------------------------------------|---------------|
| Doors, windows, plumbing fixtures, electrical fixtures, appliances,       |               |
| HVAC diffusers, light fixtures, casework units, equipment (countable)     | `EA`          |
| Walls, partitions, baseboard, casework runs, handrails, top-of-wall       |               |
| blocking, studs / plates / joists / rafters / beams / posts called out    |               |
| by length                                                                 | `LF`          |
| Floor finishes, ceiling finishes, roofing, paint, tile, sheathing,        |               |
| drywall (one face), siding, glazing area, slab area, finish areas         | `SF`          |
| Concrete, structural fill, excavation, base course, gravel                | `CY`          |
| Structural steel quoted by weight (beams, plates, decking by ton)         | `TON`         |
| Dimensional lumber quoted in board-feet on the drawing                    | `BF`          |
| Labor allowances (sub on-site time)                                       | `HR`          |

Element-type rules that catch the most common mistakes:

  * Doors / windows / fixtures / appliances / equipment — `EA`, never `LS`.
  * Walls / partitions / casework runs / baseboard — `LF`, never `LS`.
  * Floors / ceilings / paint / tile / siding / roofing / drywall area —
    `SF`, never `LS`.
  * Concrete / fill / excavation / aggregate — `CY`, never `LS`.
  * Structural steel members called out by count — `EA` with the size in
    `description`; by weight — `TON`. Never `LS`.
  * For wood-framing items in CSI 06 10 00 specifically: `LF` for studs,
    plates, joists, rafters, beams and posts; `BF` only if a board-foot
    quantity is printed on the drawing; `SF` for sheathing and deck
    boards; `EA` for hangers, connectors, hold-downs and individual
    posts called out by count.

==============================================================================
UNIT-INFERENCE FALLBACK — STILL DO NOT DEFAULT TO LS
==============================================================================

If you genuinely cannot determine a dimensional unit from the drawing or
schedule (the element is visible but its measurable quantity is not
inferable), DO NOT fall back to `unit: "LS"`. Instead, emit:

{
  "csi_division": "<best guess>",
  "csi_section":  "<best guess or null>",
  "description":  "<short description of the element>",
  "quantity":     0,
  "unit":         "EA",
  "confidence":   <low, e.g. 0.2>,
  "notes":        "unit_inference_failed: <one-line reason>"
}

Downstream pricing skips `quantity = 0` rows safely, and the
`unit_inference_failed` flag routes the line to the OPERATOR_REVIEW
queue rather than zeroing out through the unit-mismatch suppressor.
This is strictly preferred to emitting an `LS` guess for a measurable
element.

==============================================================================
GUIDANCE BY SHEET TYPE
==============================================================================

Guidance for floor plans:
  - For each room you can identify, populate `rooms` with the best area you
    can read from the drawing or schedule, and any finish notes.
  - Add `raw_takeoffs` for things you can quantify directly:
      * Total interior partition length (LF) -> CSI 09 22 16
      * Total drywall area both sides (SF)   -> CSI 09 29 00
      * Door count (EA, by type)             -> CSI 08 11 13 / 08 14 16
      * Window count or area                 -> CSI 08 50 00 / 08 51 13
      * Floor finish areas (SF) by material  -> CSI 09 65 19 / 09 68 13
      * Ceiling areas by material            -> CSI 09 51 13 / 09 29 00
      * Painted wall area (SF)               -> CSI 09 91 23

Guidance for elevations / sections / details:
  - Skip the rooms array; focus on `raw_takeoffs` for finishes (siding,
    roofing, masonry, glazing, insulation thickness etc.) and any window /
    door schedules visible.

Be conservative. A confidence of 0.9 means "directly read from the drawing".
0.5 means "estimated from scale or context". Below 0.4 means "rough guess".
