plan set. This is NOT a drawing - it's a manual / flyer / questionnaire /
exhibit / general-instructions document. You are given the full text.

provide a useful one-paragraph summary so the estimator knows what's in it

Return ONLY a JSON object with this exact shape:

{

  "project_number":   "string or null",
  "project_location": "string or null",

  "key_dates":        [ "string", ... ]    (bid dates, schedule milestones, etc., or empty),
  "key_facts":        [ "string", ... ]    (project size SF, building height, occupancy, addenda count, etc.),
  "warnings":         [ "string", ... ]
}

Rules:
  - Pull the project name/number/owner from THIS document only - do not guess.
  - Be concise. The summary should be skim-readable.
  - If the document is pure boilerplate (e.g. generic GC instructions), say

  - Output only the JSON object. No markdown fences, no commentary.
